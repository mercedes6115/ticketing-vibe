## 🎯 AI 코딩 시대의 포트폴리오 전략

> "AI 코딩 시대에는 단순 구현 능력보다 문제 정의, 설계 주도성, 공학적 검증 능력이 핵심 평가 요소가 된다."

---

### AI 시대 포트폴리오 변화 배경

| 내용 | 설명 |
| --- | --- |
| **기존 포트폴리오 한계** | 단순 CRUD 프로젝트는 AI로 쉽게 구현 가능해 차별화 어려움 |
| **면접관 평가 변화** | 코드 구현력보다 문제 정의, 설계 의도, 코드 리뷰 및 토론 능력 중시 |
| **AI 역할과 지원자 역할** | AI는 코드 구현 및 패턴 제공, 지원자는 설계, 품질 판단, 수치 측정, 결과 검증 주도 |

---

### AI 시대 포트폴리오 4대 방향성

| 방향성 | 설명 |
| --- | --- |
| **도메인 관심과 관찰력** | 프로젝트 문제 선정에 대한 본인의 관심과 주도성 표현 |
| **문제 설정 및 해결 능력** | 문제를 구체화하고 설계하여 AI와 함께 구현, 결과 검증 |
| **기술과 공학적 접근** | 설계 패턴 적용, 품질 지표 측정, AI 코드를 비판적으로 활용 |
| **검증 가능한 결과물** | 배포 URL, GitHub 코드, 설계·트레이드오프 기록, 설명 영상 등 |

## 📝 포트폴리오 주제 유형별 사례 및 전략

| 유형 | 특징 및 예시 | 전략 및 포인트 |
| --- | --- | --- |
| **1. 일상의 문제 출발** | 기차표 선착순 예매, 콘서트 예매, 데이팅 앱 추천 등 | 실제 경험 기반 문제 선정, 공학적 원리 적용, 수치 측정, 의도 명확화 |
| **2. AI 코딩 환경 실험 및 개선** | AI 할루시네이션, 버그, 컨텍스트 한계 문제 발견 및 개선 프로젝트 | AI 문제점 분석, 공학적 개선 시도, AI 활용 기법 정리 |
| **3. 빅테크 기술 발표 사례 재현** | 트위터 타임라인 캐시, 우버 실시간 유치 추적, 네이버 페이 주문 메시지 무중단 전환 등 | 발표 내용 기반 설계·구현, 장애 사례 검증, 설계 의도 재해석 |
| **4. 오픈소스 코드 분석 및 기여** | 사용 중인 작은 라이브러리 이슈 발견, 원인 분석, PR 시도 | 실무 코드 이해력, 커뮤니티 참여 의지, 분석 및 개선 과정 기록 |
| **5. 실제 사용자 있는 소규모 프로젝트** | 10명 내외 소규모 유저 대상 프로젝트 운영 | 실사용 데이터 기반 성능 및 피드백 수집, 배포·운영 경험 강조 |
| **6. 부트캠프 프로젝트 차별화** | 동일 기관·주제의 유사 프로젝트 중 차별화 필요 | 본인 기여 명확화, 기술 선택 근거 및 트레이드오프 기록, 공학적 원리 적용 |

## 💡 AI 코딩 활용법과 주도적 역할

- AI에게 맡길 일

    - 코드 구현
    - 알고리즘, 설계 패턴 기본 생성
    - 디버깅- 본인이 해야 할 일

    - 문제 정의 및 설계 주도
    - 품질 관점에서 트레이드오프 판단
    - 성능 지표 설정 및 측정
    - AI 결과물 검증 및 개선
    - 설계 의도 및 핵심 코드 설명 가능
---

## 🔍 성능 수치 및 목표 설정 방법

- **공학적 기준 설정**

    - 구체적 수치 예: 응답시간 P95 500ms 이하, TPS 특정 값, 레이턴시 수치 등- **성능 테스트 도구 활용**

    - 부하 테스트(부화 발생기 등)로 가상 사용자 생성
    - 실제 유저가 없어도 시나리오 기반 테스트 및 성능 측정 가능- **산업 표준 및 레퍼런스 참조**

    - 구글 SRE 가이드라인 등 신뢰성 있는 자료 활용

| 공학적 원리와 품질 기준, 측정 수치가 있는가? | 객관적 성능 지표 설정 및 결과 기록 |

| 다른 지원자와 구분되는 나만의 관점이 있는가? | 차별화된 시각과 주도성 표현 |

## 📢 블로그, GitHub, 유튜브 활용법

- **지속적 기록**: 단발성 결과물이 아닌 꾸준한 학습 및 개발 과정 기록
- **기술 수준과 관심사 전달**: 평소 관심 분야와 고민을 보여주는 창구
- **주의사항**

    - 얕거나 복사한 글, 일상 이야기 위주 글은 오히려 역효과
    - 방치된 블로그나 너무 외부 활동에 치중한 내용은 부정적 인상 가능
